# QA Checklist: Events + Attendance (Desktop)

## Setup
- [ ] Sign in with a staff/admin account.
- [ ] Confirm at least two people exist (People list).

## Event Creation
- [ ] Create a new event with name, start/end date, status, and registration mode.
- [ ] Optional: set parent event, empowerment, or guru fields.
- [ ] Verify the event appears in the Events list.
- [ ] Open the event detail view from the list.

## Event Days
- [ ] Verify event days are auto-created for the start/end date range.
- [ ] Add an extra day; confirm the day number and date show correctly.
- [ ] Delete a day and confirm it is removed from the list.

## Attendees
- [ ] Add an attendee with a person, registration mode, registered at/by, and optional notes.
- [ ] Edit the attendee and confirm updates persist.
- [ ] Delete the attendee and confirm it is removed.

## Attendance Grid
- [ ] Toggle an attendee to Present for a day.
- [ ] Verify the check-in time appears and checked-in-by shows the signed-in email.
- [ ] Toggle back to Absent and confirm check-in details clear.

## Guards
- [ ] After attendance exists, try editing the event start/end dates and confirm the change is rejected with an error.
